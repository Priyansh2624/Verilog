# 32-Bit Ripple Carry Adder (RCA) ASIC Implementation using OpenLane

## Overview

This project implements a **32-bit Ripple Carry Adder (RCA)** in Verilog HDL and takes it through the complete RTL-to-GDSII ASIC design flow using **OpenLane 2** and the **Sky130 PDK**.

The design is constructed hierarchically using:

- Half Adder
- Full Adder
- 32-Bit Ripple Carry Adder

The project demonstrates digital design, synthesis, physical implementation, routing, timing analysis, and GDSII generation for a combinational arithmetic circuit.

---

## Design Hierarchy

```text
Half Adder
     │
     ▼
Full Adder
     │
     ▼
32-Bit Ripple Carry Adder
```

### Half Adder

Inputs:
- A
- B

Outputs:
- Sum
- Carry

Equations:

```
Sum   = A ⊕ B
Carry = A · B
```

### Full Adder

Constructed using two Half Adders.

Inputs:
- A
- B
- Cin

Outputs:
- Sum
- Cout

Equations:

```
Sum  = A ⊕ B ⊕ Cin
Cout = AB + BCin + ACin
```

### 32-Bit RCA

The 32-bit adder is implemented by cascading 32 Full Adders.

```text
Cin → FA0 → FA1 → FA2 → ... → FA31 → Cout
```

Each stage waits for the carry generated by the previous stage, resulting in a ripple carry structure.

## Key Observations

### Advantages

- Simple architecture
- Low hardware complexity
- Smaller area compared to advanced adders
- Easy to design and verify

### Disadvantages

- Long carry propagation delay
- Poor timing performance for large bit-widths
- Not suitable for high-speed arithmetic units
